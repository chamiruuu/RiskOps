# Automated Release Publishing

One-command publish workflow for RiskOps TMS with automatic build and GitHub Release upload.

## Setup

### Prerequisites
- GitHub Personal Access Token (PAT) with `repo` and `contents` scopes

### Generate GitHub Token
1. Go to https://github.com/settings/tokens/new
2. **Token name**: `RISKOPS_PUBLISH`
3. **Expiration**: 90 days (or longer)
4. **Select scopes**:
   - ✅ `repo` (full control of private repositories)
   - ✅ `contents` (Write access to contents and metadata)
5. Click "Generate token"
6. **Copy the token** (you won't see it again)

### Set Environment Variable

**On Windows (PowerShell):**
```powershell
$env:GH_TOKEN = "your_token_here"
```

**To make it permanent (add to system):**
```powershell
[Environment]::SetEnvironmentVariable("GH_TOKEN", "your_token_here", "User")
```

**Or create a `.env` file in the project root:**
```
GH_TOKEN=your_token_here
```

Then load it before running:
```powershell
$env:GH_TOKEN = (Get-Content .env | Select-String "GH_TOKEN").ToString().Split("=")[1]
npm run publish
```

## Usage

### Step 1: Bump Version
Update the version in `package.json`:
```json
"version": "1.0.3"
```

### Step 2: Publish (One Command!)
```bash
npm run publish
```

This will:
1. ✅ Build the application (`npm run electron:build:nsis`)
2. ✅ Verify release files exist
3. ✅ Create a GitHub Release (tag: `v1.0.3`)
4. ✅ Upload 3 files:
   - `latest.yml`
   - `RiskOps-TMS-Setup-1.0.3.exe`
   - `RiskOps-TMS-Setup-1.0.3.exe.blockmap`
5. ✅ Publish the release (go live automatically)

### Step 3: Verify
The script will output:
```
🎉 Successfully published version 1.0.3!
📍 Release URL: https://github.com/chamiruuu/RiskOps/releases/tag/v1.0.3
```

Check your release on GitHub - it's live and users can now update!

## What Happens from User Side

Users with RiskOps TMS v1.0.2 installed:
1. App starts and checks GitHub Releases
2. Sees v1.0.3 is available
3. Downloads the update
4. Sees restart prompt
5. On restart, v1.0.3 is installed

## Troubleshooting

### ❌ `GH_TOKEN environment variable is not set`
Set your token:
```powershell
$env:GH_TOKEN = "your_token_here"
```

### ❌ Build fails
Make sure you've committed your changes:
```bash
git add .
git commit -m "Bump version to 1.0.3"
git push
```

### ❌ Files not found after build
Double-check the release folder has:
- `release/latest.yml`
- `release/RiskOps-TMS-Setup-{version}.exe`
- `release/RiskOps-TMS-Setup-{version}.exe.blockmap`

### ❌ GitHub Release already exists
Delete the release on GitHub first, then try again, or increment version number.

## Notes

- **Always bump version** before publishing (clients won't update if version doesn't increase)
- **Use hyphenated filenames** (they're auto-generated by electron-builder)
- **Keep old releases** on GitHub (users may need to download older versions)
- **Token security**: Don't commit `.env` files to git - add to `.gitignore`

`scripts/publish-release.js` - handles all automation
